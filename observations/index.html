Observation modules

The observations in LUMIA should be stored in an observation class, which handles storage of data in the observation space: observed and modelled concentrations, uncertainties, metadata, etc.

The observations class also handles writing and reading observations in a format that is readable by the CTM (transport package).

The observation classes themselves are stored in sub-modules:

  • lumia.observations.protocols defines templates for the observations class
  • lumia.observations.observations defines the default observations class in LUMIA. It is adapted to handling in-situ observations, and can write observation files for the pseudo-transport model (transport package) distributed with LUMIA.

The lumia.observations.observations.Observations class is made available as a top-level class, for convenience (lumia.observations.InSituObservations).

lumia.observations.observations

The module defines an Observation class, which is meant to be used in conjunction with the default lumia.models.footprints.Transport transport model class.

Structure

The Observations objects should contain two main data attributes:

  • sites is a pandas DataFrame, and contains information on observation series: site name and code, coordinates (if relevant), observation type, data provider, DOI, etc. There is no mandatory column in this table, but it should exist.
  • observations is also a pandas DataFrame and should contain:
    • the list of observations and their uncertainties
    • their corresponding index in the sites table
    • any meta-data that is relevant to the Transport object (e.g., observation time, corresponding footprint file, background concentration, observation classification, assimilation status, etc.).

Optionally, the Observations object also accepts a settings attribute, which should contain settings controlling how the observation uncertainties should be determined.

The Observations class defines the following methods:

  • I/O and type conversion methods:
    • Observations.to_hdf: saves the data to HDF format
    • Observations.save_tar: saves the data to a tar.gz format
    • Observations.to_dataframe: converts the data to a single pandas DataFrame (combines the values of the sites and observations tables).
    • Observations.from_hdf: dataclass used to generate an Observations object based on a HDF file.
    • Observations.from_tar: dataclass used to generate an Observations object based on a tar.gz file.
    • Observations.from_dataframe: dataclass used to generate an Observations object based on a single pandas DataFrame.
  • selection methods:
    • Observations.select_sites: returns an Observations object containing only the observations corresponding to a list of "sites" (as defined in the sites table).
    • Observations.select_times: returns an Observations object limited to the requested time range.
  • properties (@property):
    • mismatch: this should return a numpy NDArray containing only the model-data mismtaches (model - obs), for all observations that have both a valid (i.e. not NaN) model-data mismatch and a valid observation uncertainty.
    • sigma: this should return a numpy NDArray containing the observation uncertainties, for all observations that have both a valid (i.e. not NaN) model-data mismatch and a valid observation uncertainty.
  • Observations.calc_uncertainties: a method to estimate the observation uncertainties (i.e. combine the measurement uncertainty with an estimate of the model uncertainty).
